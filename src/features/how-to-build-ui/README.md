# how-to-build-ui

**Purpose:** Reusable UI component patterns.

## What You'll Learn
- `cn()` helper for safe className merging
- `cva` (class-variance-authority) for component variants
- Extending native HTML element types
- Compound component pattern
- Accessible components (aria, keyboard)
- Forwarding refs

## Components Built
- `Button` — with variants (primary, secondary, ghost, danger) and sizes
- `Input` — with label, error, hint
- `Card` — compound component (Card.Header, Card.Body, Card.Footer)
- `Badge` — status indicator with color variants
- `Spinner` — loading indicator
